🔤 Font Converter

Converts TTF/OTF fonts into .cpfont files the reader loads from the SD card — including kerning and ligature data extracted from the font.

1. Font files

Fallback font (fills in glyphs the main font lacks)

2. Sizes and coverage

Custom coverage
Japanese fonts with full CJK coverage rasterize ~7,000 glyphs per size and style — expect a few minutes on a phone. Glyphs are rendered by your browser's font engine, so pixel edges can differ very slightly from the desktop (FreeType) tool; metrics, kerning, and ligatures are extracted from the font file itself and match exactly.

Install on the device

The download is a zip containing .fonts/FamilyName/FamilyName_<size>.cpfont. Unzip it at the SD card root (the .fonts folder may be hidden in your file manager — that's normal), or upload via the reader's Wi-Fi web file transfer. The font then appears in the reader's font settings.

Output matches lib/EpdFont/scripts/fontconvert_sdcard.py (.cpfont v4) from the matcha-reader firmware.
